## Findings considered and rejected

- Keeping annotations as pixels and storing editable metadata separately was
  rejected because subsequent crop, blur, undo, and style changes would make
  the metadata diverge from the visible bitmap.
- Flattening annotations whenever crop or blur runs was rejected because it
  would silently remove the editability this feature is meant to provide.
- Adding a canvas framework was rejected because the repository has no build
  pipeline and the required interaction set is small enough to implement with
  the existing Canvas 2D API.
